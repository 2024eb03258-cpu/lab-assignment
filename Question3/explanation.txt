# Question 3: Links and Disk Usage - Explanations

## Overview
This question explores Linux file system concepts, including hard links, symbolic links, inodes, and disk usage analysis. These concepts are fundamental to understanding how Linux manages files and storage.

---

## Key Concepts Demonstrated

### 1. File Links in Linux
  - Hard Links: Created using `ln` command without options.
  - Share the same inode number as the original file.
  - Point directly to the same data blocks on disk.
  - Deleting the original file doesn't affect the hard link (data remains until all links are deleted).
  - Cannot link directories or cross filesystem boundaries.

  - Symbolic Links (Soft Links): Created using `ln -s`.
  - Have separate inode numbers from the original file.
  - Contain only the path to the target file.
  - Can link directories and cross filesystem boundaries.
  - Broken if the original file is deleted.

### 2. Inode System
 - Inodes store metadata about files (permissions, timestamps, data block locations).
 - Each file has a unique inode number within a filesystem
 - `ls -i` displays inode numbers.
 - Hard links share inodes; symbolic links have separate inodes.

### 3. Disk Usage Analysis
- `du` (Disk Usage): Shows space used by files and directories.
- `df` (Disk Free): Shows filesystem-level disk space availability.
- Human-readable formats (`-h` flag) for easier interpretation.
